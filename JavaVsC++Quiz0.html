Quiz 0: Java Assessment

This preliminary quiz will be used to evaluate your Java skills and understanding of rudimentary concepts. Please attempt the quiz before looking at the solution. Hints will be supplied with every question.

Animal.java 


abstract class Animal {
	Animal() {
		System.out.println("I am an Animal!");
	}
	
	abstract String Speak();
	
	static int getMemberVariable() {
		return someMemberVariable_;
	}
	
	static int someMemberVariable_;
}

Cat.java 


public class Cat extends Animal {
	Cat() {
		super();
		++someMemberVariable_;
	}
	
	@Override
	String Speak() {
		// TODO Auto-generated method stub
		return "Woof!";
	}
}

Dog.java 


public class Dog extends Animal {
	Dog() {
		super();
		++someMemberVariable_;
	}
	
	@Override
	String Speak() {
		// TODO Auto-generated method stub
		return "Meow!";
	}
}

Question 1


int a = 10, b = 0;
a += 10.9;

/*
 * What will the following statement print to the console?
 */
System.out.println("a + 10.9 = " + a + 10.9);
System.out.println("a + 10.9 = " + a);
//end of question 1
Hint
Dont forget Order of Operations and how it affects String Concatenations!

Although this is a mixed-type expression, we are moving from a high precision to a low precision (keyword: truncate).

Question 2


/*
 * What will the next 2 statements print to the console?
 */
System.out.println("b++ = " + (b++));
System.out.println("++b = " + (++b));
//end of question 2
Hint
Remember:

pre-incremented statements are evaluated before the statement is finished.

post-incremented statements are evaluated after the statement is finished.

Question 3


/*
 * Question 3:
 * What will the next 2 statements print to the console?
 */
Animal animal1 = new Cat();
System.out.println("someMemberVariable_ = " + Animal.getMemberVariable());
//end of question 3
Hint
Don't forget that the constructor of the parent class is always called before the constructor of the children. Also, note that the variable and function of the abtract class are static. Static variables and functions are associated with the class, not the instance of the class.

Question 4


/*
 * Question 4:
 * What will the next 4 statements print to the console?
 */
Animal animal2;
System.out.println("someMemberVariable_ = " + Animal.getMemberVariable());
animal2 = new Dog();
System.out.println("someMemberVariable_ = " + Animal.getMemberVariable());
//end of question 4
Hint
Continuing from Question 3, static variables and functions are associated with classes, not instances. Pay close attention to when the static variable is incremented.

Question 5

All methods in Java are potentially polymorphic. Make sure you understand where each Speak() function is being called from.
