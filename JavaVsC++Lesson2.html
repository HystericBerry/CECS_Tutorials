Lesson 2: Fundamental Data Types in Java and C++

In this lecture, we will learn how Java and C++ handles Fundamental Data Types. It is important to understand how this works for the the following reasons:

  • Optimize your program
    •

    In programming, choosing which data type you use is important to optimizing your code because it is less cpu and memory intensive. If all shopping carts were 10 feet x 10 feet x 10 feet large, everyone would have a difficult time shopping. Similarly, choosing a data type of size 8 Bytes while only using 1 bit would be unnecessary. This tends to matter especially with mission critical software - software that has a limited amount of hard drive and cpu strength (ex: airplanes). Conversely, choosing a data type that is too small is just as harmful. Imagine shopping for mattresses with a shopping cart a big as a cellphone. By attempting to "shove" large information into a tiny space, we are losing information, this is called truncating.

    Knowing how data types work also helps when debugging (fixing) errors. As stated above, attempting to shove large data into a smaller/ confined space results in a loss of data. When you lose data, you are not performing calculations on all the data that was intended. This can result in improper indexing (searching) and inaccurate answers. The idea of maintaining and assuring accuracy of data over time is called Data Integrity.

    Order of Precedence

    Order of Precedence is equivalent to the Order of Operations in Mathematics, but with additional operators. By following the "Please Excuse My Dear Aunt Sally" rule, it is important to note that exponential notation does not inheritly exist in Java and C++. Hence, users must either import libraries with an exponent function or multiply the number (N) x number of times (N*N*N*...*N). Otherwise, statements in both Java and C++ are executed from left to right. Click on the links to view order of precedence before moving on.

    Let's take a look at some examples:

    int a = 4, b = 13;
System.out.println(a + b);

    Hint
    
    There is only one operator and no left hand expression.
    Solution
    17
    int a = 4, b = 13, c;
System.out.println(a + b * b);

    Hint
    
    There are now two operators: + and *. We know from basic mathematics that the * operator takes 
precedence over the + operator.
    Solution
    173

    Strings tend to make operations more confusing:

    int a = 4, b = 13;
System.out.println("a + b = " + a + b);

    Hint
    
    There are now two operators of equivalent precedence.
    Solution
    Because there are only two operators with equal precedence, the statement is executed from left to right. This means that "a + b = " + a will be evaluated first, resulting in the String concatenation: "a + b = 4". Then, the resulting String will be concatenated again in the statement "a + b = 4" + b. Answer:
    a + b =413

    Important concepts when writing mathematical statements in programming are the Mixed-Type Expression and implicit conversion rule. Mixed-Type Expression describes the mixing of different data types in one mathematical statement. If we attempt to add an integer to a double, will the result be an integer or a double? Questions like these are answered by the concept of implicit conversion. As programmers, one of the important rules we try to uphold is the Data Integrity rule. If an integer and double resulted in an integer, this would imply we lost data. In order prevent inaccurate data, programming languages should and will always promote the resulting expression to the data type of highest precision that was used. Let's take the following generic equation: integer * character + float + double. By Order of Precedence, we have a * that is prioritized over two + operators. An integer * character will result in an integer because an integer is more precise and holds more data than a character data type (remember: all data types are technically stored/ represented as a number in Hexadecimal/binary; therefore, doing math with integers and characters is completely viable). Now, the expression has transformed from integer * character + float + double to integer + float + double. Because a float holds more data and is more precise than an integer, an integer + float results in a float data type. Finally, a double is more precise than a float. The final solution to the generic equation results in a double data type. The general result for any Argument Promotion will always be: 

    Promotion Rules:
Low Precision -> High Precision = data remains intact
High Precision -> Low Precision = truncate/ lost data

    We'll take a look at some examples to clear any confusion:

    int a = 4, b = 13;
double c = 3.1;
System.out.println(a + b + c);

    Hint
    
    Equal operators = Equal Precedence. int + int + double
    Solution
    Because all operators are of equal precedence, we read the statement from left to right. int + int results in an int and the resulting int + double results in a double. 4 + 13 + 3.1 -> 17 + 3.1 -> 20.1 Answer:
    20.1
